# SQL 编码规范

## 命名

- 表名 snake_case 复数形式 如 `users` `orders`
- 列名 snake_case 单数形式
- 主键命名为 `id` 或 `表名单数_id`
- 外键命名为引用表名单数加 `_id` 如 `user_id`
- 索引命名格式 `idx_表名_列名`
- 约束前缀 `pk_` `fk_` `uk_` `ck_`

## 格式

- 关键字大写 如 `SELECT` `FROM` `WHERE`
- 表名和列名小写
- 每个子句独占一行
- 缩进对齐子句内容
- 逗号统一放行首 保持风格一致
- 子查询额外缩进

## 查询

- `SELECT` 显式列出列名 禁用 `SELECT *`
- `JOIN` 显式写类型 `INNER JOIN` / `LEFT JOIN`
- `WHERE` 条件用参数化查询 禁止字符串拼接
- `EXISTS` 优先于 `IN` 子查询
- `UNION ALL` 优先于 `UNION` 除非需要去重
- 大数据量分页用游标分页替代 `LIMIT/OFFSET`

## 设计

- 主键用自增 ID 或 UUID 业务键做唯一约束
- 外键约束保证引用完整性
- `NOT NULL` 为默认 可空需要理由
- 适当范式化 3NF 通常足够
- 时间字段用 `TIMESTAMP WITH TIME ZONE`
- 软删除用 `deleted_at` 字段

## 索引

- 高选择性列优先建索引
- 复合索引遵循最左前缀原则
- 覆盖索引减少回表
- 避免在索引列上使用函数
- 定期分析慢查询并优化索引

## 事务

- 事务范围最小化
- 明确隔离级别
- 避免长事务锁表
- 批量操作分批提交

## 迁移

- 每次变更一个迁移文件
- 迁移可回滚 `up` / `down` 配对
- 不修改已执行的迁移
- DDL 和 DML 分开迁移
