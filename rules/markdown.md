# Markdown 书写规范

## 格式

- UTF-8 编码
- 行宽 120 字符 或不限制
- 段落之间空一行
- 文件末尾空一行
- 标题前后空一行

## 标题

- # 后加空格
- 层级递进不跳级 h1 > h2 > h3
- 一个文档一个 h1
- ATX 风格 # 不用 Setext 风格下划线

## 列表

- 无序列表用 - 不用 * 或 +
- 有序列表用 1. 自动编号
- 嵌套缩进 2 或 4 空格 保持一致
- 列表项之间空行使每项成为段落

## 代码

- 行内代码用单反引号
- 代码块用三反引号加语言标识
- 长代码块标注文件路径作为参考
- 避免截图代码 用文本代码块

## 链接与图片

- 有意义的链接文本 不用 点击这里
- 图片必须有 alt 文本
- 引用式链接用于重复 URL
- 相对路径引用仓库内资源

## 表格

- 对齐冒号标记 :--- 左对齐 :---: 居中 ---: 右对齐
- 表格内容简洁 复杂内容用列表替代
- 表头与分隔行必须

## 文档结构

- README 结构 标题 > 简介 > 安装 > 使用 > 配置 > 贡献 > 许可
- CHANGELOG 遵循 Keep a Changelog 格式
- 目录 TOC 超过 3 个章节时添加

## 规范

- 中英文之间加空格
- 专有名词保持原始大小写 如 JavaScript GitHub
- 避免 HTML 标签 除非 Markdown 无法实现
