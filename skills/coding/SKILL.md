# 主要基本规则

- 只能改必要的代码 也就是在更改最少代码的情况下实现用户的目标
- 尽可能使单个文件中的代码功能单一 只执行某个特定功能 减少单个文件行数
- 请勿在任何代码中添加任何注释
- 请注意代码的可读性 有合理换行和缩进
- 在每个块之间有一个空行
